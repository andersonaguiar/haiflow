**haiflow** (**h**ooks · **ai** · **flow**) is an HTTP orchestrator for [Claude Code](https://docs.anthropic.com/en/docs/claude-code). It wraps Claude Code in `tmux` sessions and exposes a REST API so you can trigger prompts, queue work, stream responses, and chain agents together, all from any HTTP client.

Anything you can do in Claude Code (code generation, refactoring, bug triage, daily reports) you can automate over HTTP with cron, webhooks, or any tool that can make a request.

## Why haiflow

<Note>
**Why not just the Claude API?** Claude Code already includes tool use, file access, git integration, and your custom skills. haiflow lets you automate all of that over HTTP without paying per-token API costs, because it runs the local `claude` CLI and reuses your existing Claude Code login. Your `HAIFLOW_API_KEY` is a secret you choose to protect the server, not a paid credential.
</Note>

- **No per-token costs.** Tasks run on your flat Claude Code subscription. haiflow even reports the API cost you avoided (see [Usage & savings](/guides/usage-and-savings)).
- **Full agent capability.** File access, shell, `git`, `gh`, and your skills, not just text completion.
- **Drive it from anywhere.** A plain REST API plus a built-in [dashboard](/guides/dashboard), an [MCP server](/integrations/mcp), and a [CLI](/guides/cli).
- **Built for automation.** A FIFO queue with priority, dedup, scheduling, fire-and-forget jobs, and event pipelines.

## How it works

```
POST /trigger ───┐
                 │        ┌────────────────┐
             ┌───▼───┐    │  tmux session  │
             │ Queue ├───>│   (claude)     │
             │ (FIFO)│    └───────┬────────┘
             └───────┘            │
                           hooks fire on
                           session events
                                  │
                          ┌───────▼────────┐
                          │    Responses   │
                          └───────┬────────┘
                                  │
GET /responses/:id <──────────────┤
                                  │
GET /responses/:id/stream <───────┘  (SSE)
```

1. `POST /session/start` spawns Claude in a detached tmux session.
2. `POST /trigger` sends a prompt (or queues it when busy) and assigns a task id.
3. Claude Code hooks forward lifecycle events (start, prompt, stop, end) back to the haiflow server.
4. On completion, the server extracts the assistant messages from the session transcript and saves them keyed by task id.
5. `GET /responses/:id` returns the result, or you stream it live with `GET /responses/:id/stream`.

## Platform support

macOS and Linux only. Windows is not supported yet, because haiflow depends on `tmux` and POSIX shell scripts.
